# RAG 模式使用示例

本文档提供 PaiSmart 两种 RAG 模式的实际使用示例和对比。

---

## 场景 1：企业内部政策咨询

### 用户问题
```
公司的年假制度是什么？
```

### 严格模式 (Strict Mode)

**配置**:
```yaml
llm:
  rag_mode: "strict"
elasticsearch:
  min_relevance_score: 0.6
```

**情况 A - 知识库有内容**:
```
根据公司人力资源手册，年假制度如下：

1. 工作满1年：5天年假
2. 工作满5年：10天年假
3. 工作满10年：15天年假

年假需提前3天申请，批准后方可休假。

(来源#1: 员工手册-2024版.pdf)
```

**情况 B - 知识库无内容**:
```
抱歉，知识库中暂无相关信息，无法回答您的问题。
```

---

### 灵活模式 (Flexible Mode)

**配置**:
```yaml
llm:
  rag_mode: "flexible"
elasticsearch:
  min_relevance_score: 0.4
```

**情况 A - 知识库有内容**:
```
根据公司人力资源手册，年假制度如下：

1. 工作满1年：5天年假
2. 工作满5年：10天年假
3. 工作满10年：15天年假

年假需提前3天申请，批准后方可休假。

(来源#1: 员工手册-2024版.pdf)
```

**情况 B - 知识库无内容**:
```
【注意：以下回答基于AI通用知识，非来自知识库】

一般企业的年假制度通常遵循《职工带薪年休假条例》：

1. 工作满1年不满10年：5天年假
2. 工作满10年不满20年：10天年假
3. 工作满20年：15天年假

但具体制度因公司而异，建议咨询人力资源部门获取准确信息。
```

## 场景 2：技术文档查询

### 用户问题
```
如何配置 Redis 连接池？
```

### 严格模式响应

**知识库有内容**:
```
在本系统中配置 Redis 连接池的方法如下：

1. 编辑 configs/config.yaml 文件
2. 修改 redis 配置段：
   ```
   redis:
     addr: "127.0.0.1:6379"
     password: "your-password"
     db: 0
   ```

系统会自动初始化连接池，默认参数：
- 最大连接数：10
- 空闲超时：5分钟

(来源#2: 系统配置指南.md)
```

**知识库无内容**:
```
抱歉，知识库中暂无相关信息，无法回答您的问题。
```

---

### 灵活模式响应

**知识库有内容**: (同严格模式)

**知识库无内容**:
```
【注意：以下回答基于AI通用知识，非来自知识库】

配置 Redis 连接池通常需要设置以下参数：

1. **基础连接信息**：
   - 地址 (host:port)
   - 密码 (password)
   - 数据库编号 (db)

2. **连接池参数**：
   - MaxIdle: 最大空闲连接数
   - MaxActive: 最大活跃连接数
   - IdleTimeout: 空闲连接超时时间
   - Wait: 连接池满时是否等待

具体配置方式取决于使用的 Redis 客户端库。
```

## 场景 4：法律合规咨询

### 用户问题
```
员工竞业限制协议的有效期是多久？
```

### 推荐配置：严格模式

**配置**:
```yaml
llm:
  rag_mode: "strict"
elasticsearch:
  min_relevance_score: 0.7  # 高阈值，确保精准
```

**原因**：
- 法律信息需要准确引用
- 避免 AI 提供不准确的法律建议
- 确保答案可追溯到具体文档

**知识库有内容**:
```
根据公司《竞业限制管理办法》第5条规定：

竞业限制的期限不得超过2年，具体期限在协议中约定。
竞业限制期间，公司按月支付补偿金，标准不低于劳动合同
解除前12个月平均工资的30%。

竞业限制范围包括：
1. 同类产品或服务的竞争企业
2. 自行开展与公司业务竞争的经营活动

(来源#4: 竞业限制管理办法-2024版.pdf)
```

**知识库无内容**:
```
抱歉，知识库中暂无相关信息，无法回答您的问题。

建议您：
1. 咨询公司法务部门
2. 查阅相关法律文件
3. 寻求专业律师意见
```

## 模式选择决策树

```
用户提问
    |
    ├─ 需要准确引用来源？
    |       |
    |       ├─ 是 → 严格模式 (Strict)
    |       |      - 法律文档
    |       |      - 企业政策
    |       |      - 技术规范
    |       |
    |       └─ 否 → 继续判断
    |
    └─ 需要兜底回答？
            |
            ├─ 是 → 灵活模式 (Flexible)
            |      - 客服场景
            |      - 通用咨询
            |      - 混合知识
            |
            └─ 否 → 严格模式 (Strict)
                   - 专业领域
                   - 合规要求
```

## 最佳实践建议

### 1. 根据业务场景选择模式

```yaml
# 企业内部知识库 - 严格模式
llm:
  rag_mode: "strict"
elasticsearch:
  min_relevance_score: 0.6

# 客服机器人 - 灵活模式
llm:
  rag_mode: "flexible"
elasticsearch:
  min_relevance_score: 0.4
```

### 2. 监控和优化

- 记录"无检索结果"的问题
- 定期补充知识库内容
- 根据用户反馈调整阈值

### 3. 混合使用策略

考虑为不同类型的问题配置不同的模式（需要自定义开发）：

```go
// 伪代码示例
if isLegalQuestion(query) {
    ragMode = "strict"
} else if isGeneralQuestion(query) {
    ragMode = "flexible"
}
```

## 常见问题

### Q: 灵活模式下如何避免 AI 提供错误信息？

**A**: 
1. 设置合理的相关性阈值（0.4-0.5）
2. 定期审查 AI 生成的回答
3. 在前端明确显示"非知识库内容"标注
4. 收集用户反馈，持续优化

### Q: 严格模式下"无法回答"太多怎么办？

**A**:
1. 降低相关性阈值（从 0.7 降到 0.5）
2. 优化知识库内容的覆盖度
3. 改进文档的分块策略
4. 考虑切换到灵活模式

### Q: 能否运行时切换模式？

**A**: 当前版本需要修改配置文件并重启服务。如需运行时切换，可以考虑：
1. 使用环境变量 + 热加载
2. 提供管理后台配置界面
3. 根据用户角色自动选择模式
